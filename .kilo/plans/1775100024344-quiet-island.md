# Plano: Redirecionar Link de Aceite para Contrato (em vez de Sucesso-Cadastro)

## Contexto

Atualmente, quando um futuro tomador interrompe o cadastro após "Aceitar e Enviar" (ou seja, não aceita o contrato), o suporte pode enviar um link para que ele retome o fluxo. Porém, o link está direcionando para `/sucesso-cadastro`, que contém lógica legada de plano/plano personalizado e o fluxo de aceite não está funcionando corretamente.

**Fluxo atual (problema):**

1. Suporte gera link em `/suporte` → `buildAceiteUrl()` → `/sucesso-cadastro?id=X&contrato_id=Y`
2. Usuário clica no link → vê página com lógica de pagamento legada (não focada no aceite)
3. Fluxo confuso - a página tenta carregar dados de plano/sessão que não existem para novos cadastros

**Fluxo desejado:**

1. Suporte gera link → direciona para tela de aceite do contrato
2. Usuário aceita o contrato → registro no banco (`contratos.aceito = true`)
3. Avançar para fluxo de liberação de login e senha → tela `/boas-vindas` (credenciais) → `/login`

## Mudanças Necessárias

**Remover:**

**Resultado esperado:** Uma página limpa que:

1. Recebe `id` e `contrato_id` da URL
2. Abre o `ModalContrato` automaticamente
3. Após aceite, redireciona para `/boas-vindas` (via API response)

## Fluxo Final

1. **Cadastro inicial:** Usuário preenche formulário → submit → redirect para `/sucesso-cadastro?id=X&contrato_id=Y`
2. **Aceite do contrato:** Página abre `ModalContrato` automaticamente → usuário lê → marca checkbox → clica "Aceitar Contrato e Liberar Acesso"
3. **Backend processa:**
   - `contratos.aceito = true`
   - `criarContaResponsavel()` cria conta na tabela `usuarios` + senha em `entidades_senhas`/`clinicas_senhas`
   - `entidades.ativa = true` ou `clinicas.ativa = true`
4. **Redirecionamento:** API retorna `boasVindasUrl` → `/boas-vindas?tomador_id=X&login=CPF&senha=SENHA`
5. **Tela de credenciais:** Usuário vê suas credenciais (login + senha) com opção de copiar
6. **Login:** Após countdown ou clique, usuário vai para `/login` e faz login com CPF + últimos 6 dígitos do CNPJ como senha (ao logar, será solicitado troca de senha)

## Caso do Suporte

Quando o suporte precisa reenviar o link:

1. Suporte acessa `/suporte` → seção "Pré-cadastros"
2. Clica "Gerar Link" → copia URL `/sucesso-cadastro?id=X&contrato_id=Y`
3. Envia por e-mail/WhatsApp para o tomador
4. Tomador clica → vê tela de aceite → aceita → redirecionado para `/boas-vindas` (credenciais) → `/login`
