# Design Context — QWork v2

## Users & Contexts

### Dois contextos distintos:
1. **Funcionários** (mobile-first): Avaliação COPSOQ III com fluxo linear 37 questões, respostas anônimas, conclusão com modal de sucesso
2. **Profissionais de Saúde/RH/Gestores/Emissores** (desktop-first): Gestão de lotes, empresas, laudos, pagamentos, emissão de documentos legais

## Brand Personality

**QWork é:** Confiável, Técnico, Eficiente

**Não é:** Genérico, colorido, lúdico, especulativo

**Pilares:**
- Conformidade legal (NR-1, LGPD)
- Auditoria completa com SHA-256
- Transparência de processo
- Segurança de dados

## Aesthetic Direction

### Color Palette
- **Primary (Charcoal):** `#2D2D2D` — confiança, solidez
- **Accent (Verde):** `#9ACD32` — ação, validação, progresso
- **Neutrals:** Tailwind gray-* spectrum (50–950)
- **Warning/Alert:** Amber/Red para erros, conforme contexto
- **Orange `#FF6B00`:** APENAS templates PDF/print (laudo aprovado), NUNCA em UI live

### Typography
- **Sans-serif:** Geist Sans (já configurada no projeto)
- **Monospace:** Geist Mono (para dados técnicos: CPF, hash SHA-256, IDs)
- **Fallback:** Tailwind defaults
- **Font Loading:** System fonts, sem extras de rede

### Icons
- **Provider:** Lucide React
- **Rule:** Zero emojis em código (`⚠️📄⚖️🔐👁️🔗✓📋🚀💳❌🟡🟢` → ícones sempre)
- **Size:** 16px (w-4 h-4) para inline, 24px+ para destaque

### Reference & Anti-reference
- **Reference:** qwork.app.br — clean, profissional, legível, compliance-forward
- **Anti-reference:** Purple/blue AI dashboards, gradient text, emoji-heavy UI, dark mode

## Design Principles

1. **Clareza acima de estética** — compliance requer legibilidade imediata
2. **Mobile-first para avaliação, desktop-first para gestão** — dois públicos, dois layouts
3. **Hierarquia visual clara:** 1 ação primária per tela, secundárias discretas
4. **Erros devem ensinar** — mensagens técnicas → conversão para português amigável
5. **Zero emojis no código** — Lucide icons sempre
6. **Touch targets:** mínimo 44×44px em mobile
7. **Motion:** Spring physics (stiffness: 100, damping: 20), nunca linear easing
8. **Dados sensíveis:** Mascarar CPF, logs sem senha/token, LGPD compliance na UI

## Layout & Spacing

- **Safe area:** iOS notch/Dynamic Island via `env(safe-area-inset-*)`
- **Breakpoints:** xs (475px), sm (640px), md (768px), lg (1024px), xl (1280px)
- **Grid base:** 4px (Tailwind default)
- **Cards:** Border radius 8–12px, shadow-md para profundidade discreta
- **Padding/Margin:** Consistente com Tailwind scale (16px blocks preferred)

## Constraints (Absolute)

| Constraint | Razão |
|---|---|
| Sem dark mode | Light only (usuários em contexto de saúde ocupacional precisam de contraste claro) |
| Sem novos componentes centrais | Button, Badge → corrigir inline, não criar biblioteca |
| Sem PDF generation changes | Puppeteer + `gerarLaudoCompletoEmitirPDF` intactos |
| Sem state machine changes | Lotes, avaliações, pagamentos via triggers DB |
| Sem migrations | Pure frontend/UI only |

## Implementação & Validação

- **TypeScript strict mode:** Obrigatório
- **ESLint + Prettier:** `pnpm lint` antes de commit
- **Testing:** Vitest (unit) + Cypress (e2e) — regressão zero
- **Build:** `pnpm build` sem warnings
- **Type-check:** `pnpm type-check` clean
